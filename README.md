## The Problem

在控制型关系中，最致命的事情从来不是伤害本身。是**看不见**。

当一个人还在问「对方是不是有问题」的时候，自尊已被系统性蚕食。当还在问「是不是我太敏感」的时候，经济、社会网络、自我判断力已被层层拆除。当终于确认「我该离开」的时候 — 已经被困在一个没有能力离开的系统里。

**大多数清醒来得太晚。不是因为被困的人不够聪明。是因为没有工具。**

传统的情感建议框架能给出的通常是：

- 「对方很坏，离开TA」 — 但走不了，这句建议变成了自我审判
- 「你需要变强大」 — 但不知道变强的第一步是什么
- 「对方可能有人格障碍」 — 标签能解释但不能改变任何事情

**需要的不只是意见。需要的是分析框架 — 能让当事人自己看见系统如何运作。**

### 退出可行性 / Exit Feasibility

```
退出可行性 = ( 经济独立 + 社会支持 + 心理独立 + 法律保护 ) / 安全风险
             每项 1-10 评分，安全风险底线为 1（非零）
```

| 得分 | 含义 | 行动 |
|------|------|------|
| **< 1** | 基础资产为零 | 先建立经济/社会/安全基础 |
| **1 – 1.5** | 条件形成中 | 找最弱维度，集中补强 |
| **1.5 – 2** | 可以计划 | 制定具体时间线 |
| **> 2** | 条件已具备 | 唯一阻碍是心理层——直面它 |

> 即使得分 > 2 仍感到害怕，说明有公式未捕获的壁垒。破幻会继续追问那个真正的分母。

## The Core Framework

### I. 退出不等式 / The Exit Inequality

控制者全部的操作，都在操纵这个不等式的两边：

```
留下的效用 = 已知的经济收入 + 社会身份 + 幻觉 + 间歇性温暖 − 痛苦 − 恐惧
离开的效用 = 未知的自由 + 安全 + 自我尊重 − 不确定性 − 审判 − 报复风险
```

只要离开的效用 < 留下的效用，就被困住。不需要锁链。

### II. 三层控制模型 / Three-Layer Control Model

```
第一层：物理控制   不能走    (最显眼)
第二层：社会控制   没地方去  (最被合理化)
第三层：心理控制   不想走    (最致命 — 声音已住进大脑)
```

### III. 五个角色模型 / Five Character Archetypes

每个模型都是功能化的、不依赖社会身份的分析工具：

| Model | Core Question |
|-------|--------------|
| **Self-Deceiving Controller** | How can harm and rationality coexist in one person? |
| **Cycle Originator** | Who produces the controller? How does the pattern transmit? |
| **Weaponized Virtue** | Why are the best people trapped the longest? |
| **System Collaborator** | Why do people who seem to help you actually lock the door? |
| **Control Conduit** | After the relationship ends, how does control continue? |

## What Makes This Different

### 1. 基于第一性原理的分析，不是情感建议

大多数关系分析框架建立在「谁对谁错」的道德判断上。这套系统建立在几个不可辩驳的底层原理上：
- 退出不等式（留/开的效用计算）
- 行为 vs 话语的严格分离（话语的权重低于行为 100 倍）
- 激励结构分析（「告诉我激励，我告诉你结果」）
- 退出可行性乘以资源的量化公式

### 2. 不是给答案，是给框架

传统情感建议说「你应该离开」 — 用户收到的是结论，不是工具。这套系统给的是「17 个结构化问题 + 5 步分析协议 + 自我评估维度 + 行动优先级框架」。用户学会了这个框架，就再也不会被困住。

### 3. 角色模型独立于性别、文化、身份

五个角色模型不依赖「丈夫/妻子/婆婆」这样的社会角色。它们可以出现在任何关系类型中 — 伴侣、家庭、职场、组织。跨文化、跨场景适用。

### 4. 从真实故事蒸馏出通用模型

系统的参数不是来自理论推演 — 是来自真实经历中的行为模式反复提取。经过匿名化、一般化处理后，形成功能化的分析工具。每一行都有真实的根基。

### 5. 安全先于分析

内置安全边界（3 个危急信号触发器 + 5 个高风险触发器 + 数字安全提示）。没有哪个分析比安全更重要。

### 6. 双语 · 中英双语同文

所有技能文件是中英双语交错编写，全球用户可以用同一份文档获取同等质量的分析。

## How to Use

This system is built as a Claude Code skill set. You can:

1. Place these `.md` files in your `.claude/skills/` directory
2. Invoke with `/dispel` or via trigger words in conversation
3. Each skill can be used independently or as part of the full analysis pipeline

**Quick scan (2 min):** Ask Q1 (specific behavior), Q6 (economic independence), Q9 (what you've tried) → get a directional answer.

**Full analysis (bounded conversation):** Execute all 17 questions across 4 tiers → run 5-step analysis → get output in mirror/X-ray/wake-up mode.
